# ds-create-component Skill

**Create new web components in the Helvetia Design System.**

This repository skill generates complete Stencil components with shadow DOM, TypeScript interfaces, SCSS with design tokens, and visual HTML test files.

## Usage

```bash
/ds-create-component
```

The skill guides you through an interactive questionnaire, then generates all necessary files in one pass.

## Questionnaire

The skill asks:

1. **Component name** вЂ” e.g., `button`, `card`, `modal`
2. **Purpose** вЂ” What does it do?
3. **Migration?** вЂ” From old design system? (auto-extracts props/events)
4. **Props** вЂ” Define component properties
5. **Events** вЂ” Define custom events
6. **Subcomponents?** вЂ” Child components? (e.g., `tab` inside `tabs`)
7. **Variants** вЂ” Visual variants (e.g., primary, secondary, danger)

## Migration Handling

When migrating from the old design system:

- вњ… Auto-extracts props and events from the old component
- вњ… Pre-fills questionnaire with old interface
- вњ… Flags a11y/SEO breaking changes (generates accessible version anyway)
- вњ… Suggests token migration path

Example:

```
вљ пёЏ BREAKING CHANGE (a11y):
   Old: <ds-button label="text"></ds-button>
   New: <ds-button>Click me</ds-button>

   Reason: Light DOM content improves accessibility and SEO.
```

## What's NOT Included

This skill only generates components. For other tasks:

- **Tests** вЂ” Use `/ds-sync-component-tests`
- **Storybook stories** вЂ” Create separately
- **Type definitions** вЂ” Auto-generated by `pnpm build`

## Next Steps

After the skill generates files:

1. Review the generated `.tsx`, `.interfaces.ts`, `.host.scss`, and `.visual.html`
2. Refine the `render()` method and add component logic
3. Update SCSS variants and add real styling
4. Run `pnpm play` to see your component
5. Create tests with `/ds-sync-component-tests`
