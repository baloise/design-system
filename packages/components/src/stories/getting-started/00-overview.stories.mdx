# Getting Started

In this section you find a small introduction about our toolchain.

## Documentation

To learn how the documentation of the Baloise Design System works, visit the page [How to use Storybook](?path=/docs/getting-started-how-to-use-storybook--page).

<div className="link-list" style={{ display: 'block' }}>

    <span>
      <strong>How to use Storybook</strong>
      Storybook is a tool for UI development. It makes development faster and easier by isolating components. This allows
      you to work on one component at a time. You can develop entire UIs without needing to start up a complex dev stack,
      force certain data into your database, or navigate around your application.
    </span>

## Testing

We test our web applications using the tool [Cypress](https://www.cypress.io/).

<div className="link-list" style={{ display: 'block' }}>
  <a className="link-item" href="?path=/docs/getting-started-testing--page">
    <img src={CypressLogo} alt="cypress" />
    <span>
      <strong>E-2-E Testing with Cypress</strong>
      The Baloise Design System provides a collection of custom cypress commands for our components. Furthermore some basic
      cypress commands like "should" or "click" have been overwritten to work with our components.
    </span>
  </a>
</div>
