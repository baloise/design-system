## Accessibility

### HTML / CSS only

Use the native `<details>` and `<summary>` elements which provide built-in keyboard interaction and screen reader support. The component wraps these in a semantic heading element.

<Markdown>
  {`
| Requirement          | How to implement                                                                                    |
| -------------------- | --------------------------------------------------------------------------------------------------- |
| Heading hierarchy    | Place the accordion inside a heading level (\`<h1>\`–\`<h6>\`) that respects page structure         |
| Summary semantics    | Use the \`<summary>\` element as the clickable trigger; it is keyboard-accessible by default        |
| Content containment  | Place expandable content inside the \`<details>\` element, after the \`<summary>\`                  |
| Group semantics      | Use the \`name\` attribute on \`<details>\` to group accordions so only one can be open at a time   |
| Nested accordions    | Nest \`<details>\` elements for multi-level expansion; browser and AT handle focus correctly        |
| Interactive elements | Buttons and links inside \`<summary>\` are keyboard-accessible; clicks on them do not toggle        |
`}
</Markdown>

> **Note:** The native `<details>` / `<summary>` pair handles all ARIA attributes automatically.
> Do not add `aria-expanded`, `aria-controls`, or `role="button"` — the browser provides these.

### Web component

The `ds-accordion` element wraps the native `<details>` behavior and provides additional features:

- **Heading wrapper** — renders the summary inside a configurable heading level (`<h1>`–`<h6>`) via the `summaryLevel` prop; defaults to `<h3>`
- **`aria-expanded`** — automatically kept in sync with the `open` state on the summary button
- **`aria-controls`** — set to the content region ID, linking the button to the expandable content
- **Group management** — accordions with the same `group` prop name auto-close siblings when one opens; consumers can also use the native `name` attribute for browser-native grouping
- **Button variant** — when `button={true}`, the trigger is styled as a `ds-button` instead of a summary element; all ARIA attributes are still managed
- **Keyboard navigation** — Space/Enter toggles the accordion; native browser behavior for `<details>` element
- **Event emission** — `dsToggle`, `dsOpened`, and `dsClosed` events are fired on state change; IDs and group names are included in event details for consumer logic
