## When to Use

- Select multiple independent options (preferences, interests, features)
- Accept terms and conditions or legal agreements
- Toggle features or permissions on/off
- Filter content by multiple categories or tags
- Enable users to select any combination of options

## When NOT to Use

- For mutually exclusive choices — use radio buttons instead
- When users must select exactly one option — radio buttons are clearer
- For on/off switching — use toggle component instead
- When there are only two options — consider using a single checkbox or toggle
- For non-binary yes/no questions — consider other form controls

## Do's and Don'ts

### Clear Checkbox Labels

Always provide clear, descriptive labels for each checkbox. Never rely on ambiguous labels.
