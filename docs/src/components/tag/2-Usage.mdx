## When to Use

- Display categories or classifications for content
- Show skill tags or technology labels
- Indicate status or priority levels
- Filter or organize items by attribute
- Display removable tokens in an input field
- Show badges or labels for user-generated content

## When NOT to Use

- For standalone buttons or navigation elements — use the button component instead
- To display primary action calls-to-action — buttons are more prominent
- For important alerts or critical information — use notification or alert components
- As large block-level containers — tags are intended to be compact inline elements

## Do's and Don'ts

### Tag Content and Length

Keep tag content short and concise. Use single words or short phrases that quickly convey meaning.

### Color and Semantic Meaning

Use different tag colors to convey semantic meaning or category. Be consistent across your application.
