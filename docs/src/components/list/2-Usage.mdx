## When to Use

- Display a series of related items or steps
- Create navigation menus or breadcrumbs
- Show search results or filtered content
- Organize instructions or steps in sequence
- Display related links or resources
- Present hierarchical information with nesting

## When NOT to Use

- For tabular data with multiple columns — use a table component instead
- As a replacement for proper content hierarchy — lists supplement structure, not define it
- For single items or pairs of items — simple dividers are clearer
- When items don't have a logical grouping — use individual components instead

## Do's and Don'ts

### List Item Content

Keep list items concise but descriptive. Use consistent formatting across items.
