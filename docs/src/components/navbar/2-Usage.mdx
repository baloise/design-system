## When to Use

- Creating the primary navigation header for your application
- Providing site branding and logo placement
- Organizing main navigation links (menu-start, menu-end slots)
- Adding action buttons or user account controls
- Implementing responsive mobile menu with sidebar drawer
- Building accessible navigation with semantic HTML (`<nav>`, `<aside>`, `<dialog>`)

## When NOT to Use

- For secondary or in-page navigation — use breadcrumbs or tabs instead
- For long lists of navigation items without grouping — consider subnav or mega menu patterns
- For mobile-only navigation — navbar is responsive and works on all screen sizes
- When you need custom animations — the navbar has built-in animation handling
