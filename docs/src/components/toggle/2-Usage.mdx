## When to Use

- Toggle features on/off (e.g., notifications, settings)
- Enable/disable single options in settings
- Switch between two mutually exclusive states
- Binary yes/no form questions
- Feature flags or preference toggles

## When NOT to Use

- For multiple choice (more than 2 options) — use radio buttons or dropdown
- For independent selections (multiple options) — use checkboxes
- For boolean values in tables — consider using badge or status indicators
- For lengthy explanations — toggle's compact nature may not suit explanation
- For critical actions — consider a confirmation dialog

## Do's and Don'ts

### Clear Labels

Always pair toggles with descriptive labels. Users need to know what is being toggled.
