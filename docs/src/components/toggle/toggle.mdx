Apply the `disabled` attribute to prevent user interaction. The toggle remains visible but cannot be changed.

<BasicStoryTabs tag="toggle" htmlStory={ToggleStories.DisabledHtml} webComponentStory={ToggleStories.Disabled} index={1} noGuide />

<StoryHeading of={ToggleStories.Invalid}></StoryHeading>

Apply the `invalid` attribute to indicate a validation error, for example when a required toggle has not been checked before form submission.

<BasicStoryTabs tag="toggle" htmlStory={ToggleStories.InvalidHtml} webComponentStory={ToggleStories.Invalid} index={1} noGuide />

{/* ------------------------------------------------------ */}

<StoryHeading of={ToggleStories.Field}></StoryHeading>

Wrap the toggle in a `fieldset.field` to add a label and help text. This gives the control the full field layout consistent with other form inputs.

<BasicStoryTabs tag="toggle" htmlStory={ToggleStories.FieldHtml} webComponentStory={ToggleStories.Field} index={1} noGuide />

{/* ------------------------------------------------------ */}

<StoryHeading of={ToggleStories.Form}></StoryHeading>

Use `ds-toggle` inside a `<form>` element. The `name` and `value` attributes control what is submitted. The form resets cleanly when a reset button is triggered.
