<AngularFramework>

Use the `BalSnackbarService` to create new notices.

```typescript
// app.component.ts
import { Component } from '@angular/core'
import { SnackbarService } from '@baloise/ds-angular'

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
})
export class AppComponent {
  mySnackbar?: HTMLBalSnackbarElement

  constructor(public snackbar: BalSnackbarService) {}

  onButtonClick() {
    this.mySnackbar = this.snackbar.create({
      subject: 'Subject',
      message: 'Lorem ipsum dolor sit amet, consectetur adipisicing elit.',
      icon: 'info-circle',
    })
  }

  async onSecondButtonClick() {
    await this.mySnackbar.close()
  }
}
```

## Accessibility

### HTML / CSS only

Snackbars are **polite live region notifications** meant to be dynamically injected. They inform users of events with optional actions, and remain visible until manually dismissed or auto-dismiss expires. Always include descriptive text and provide contextual action labels.

<Markdown>
  {`
| Requirement          | How to implement                                                                           |
| -------------------- | ------------------------------------ |
| Notification text    | Provide a \`heading\` and \`message\` prop or slot content so users understand the event    |
| Action button        | Use the \`action\` and \`actionHref\` props; provide descriptive action text (not "Click")  |
| Close button         | Snackbars default to \`closable={true}\`; set \`closable={false}\` only if auto-dismiss set |
| Icon semantics       | Icon is decorative; color communicates severity (danger, warning, success, info, base)     |
| Mobile layout        | On mobile, snackbar is collapsed by default; users expand via a button; full content shown  |
| Custom SVG           | Use the \`svg\` prop to provide custom icon content; avoid images without text labels       |
`}
</Markdown>

> **Note:** Snackbars use `role="status"` and `aria-live="polite"`, making them wait for a pause in speech before announcing.
> Snackbars are best for informative, non-critical notifications. Always pair icon colors with text so color is not the only cue for meaning.

### Web component

The `ds-snackbar` element is a **polite live region notification**:

- **`role="status"`** — automatically set; screen reader announces when there is a pause in speech
- **`aria-live="polite"`** — waits for natural speaking pause; no interruption
- **`aria-atomic="true"`** — entire snackbar content is announced as one unit
- **Heading as `<h2>`** — if a `heading` prop is provided, it renders as an `<h2>` for document structure
- **Icon** — automatically derived from `color` prop (info → bell, success → check, warning/danger → alert); custom icons via `icon` prop
- **Action button** — optional `ds-button` with descriptive label; triggered via `dsActionClick` event
- **Close button** — enabled by default; emits `dsCloseClick` event; parent must handle removal
- **Mobile responsive** — on small screens, snackbar collapses to an expandable button; expands on click to show full content
- **Custom SVG** — supports inline `svg` prop for brand/custom icons; animated icons optional
