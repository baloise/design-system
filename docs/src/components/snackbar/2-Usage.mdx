## When to Use

- Display persistent notifications that remain visible until the user dismisses them
- Show feedback for user actions that have completed successfully or with warnings
- Provide important information that users should be able to review at their own pace
- Display multiple notifications stacked on top of each other

## When NOT to Use

- For time-sensitive notifications that disappear automatically — use **Toast** instead
- For critical errors that require immediate user attention — use **Alert** in a modal dialog
- For inline validation feedback on form fields — use field-level error messages
