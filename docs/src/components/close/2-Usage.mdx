## When to Use

- Dismiss notifications or alerts
- Close modals or dialogs
- Remove tags from a tag list or input
- Close collapsible panels or popovers
- Remove items from selectable lists
- Dismiss temporary UI elements

## When NOT to Use

- As a generic button — use the button component instead
- For primary navigation or actions — close buttons should be secondary
- Without clear intent about what will be dismissed
- On elements that can't be dismissed or hidden
- Without proper labeling and accessibility attributes

## Do's and Don'ts

### Close Button Placement

Place close buttons in logical, consistent locations. Top-right is the standard for modals and cards.

### Close Button Accessibility

Always provide an accessible label. The X symbol alone is not sufficient.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Close button with aria-label',
      content: (
        <>
          <ds-close aria-label="Close dialog"></ds-close>
          <div style={{ textAlign: 'left', marginTop: '12px' }}>
            <Code noPreview code={'<ds-close aria-label="Close dialog"></ds-close>'} />
          </div>
        </>
      ),
      description: 'Screen readers announce button purpose',
    },
    {
      type: 'incorrect',
      title: 'Incorrect: Close button without label',
      content: (
        <ds-close></ds-close>
      ),
      description: 'Screen readers won\'t announce purpose',
    },
  ]}
/>
