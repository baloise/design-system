## When to Use

- Organising related content into distinct sections within a single page view
- Allowing users to switch between multiple views without leaving the current context
- Reducing vertical scroll by grouping content that doesn't need to be visible simultaneously
- As a navigation aid when linking to page sections via anchor links (navigation variant)

## When NOT to Use

- When content needs to be compared side by side — use a layout instead
- When there are fewer than two tabs — just show the content directly
- When tab content is deeply nested or requires its own navigation
- When the sequence of content matters — use a stepper or accordion instead

### Tab Labels

### Panels vs. Navigation

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Panels for toggled content',
      content: (

          <ds-tab-panel for="a">Content A</ds-tab-panel>
          <ds-tab-panel for="b">Content B</ds-tab-panel>
        </ds-tabs>
      ),
      description: 'Use ds-tab-panel children when the content is managed in place.',
    },
    {
      type: 'correct',
      title: 'Correct: Navigation for anchor links',
      content: (
        <ds-tabs label="Page sections">
          <ds-tab name="overview"><a href="#overview" aria-current="page">Overview</a></ds-tab>
          <ds-tab name="details"><a href="#details">Details</a></ds-tab>
        </ds-tabs>
      ),
      description: 'Use anchor link children (without ds-tab-panel) when tabs navigate to page sections.',
    },
  ]}
/>
