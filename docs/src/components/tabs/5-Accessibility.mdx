## WCAG 2.2 AA Compliance

The tabs component implements patterns that meet WCAG 2.2 Level AA standards:

- **1.3.1 Info and Relationships** — The tablist uses `role="tablist"`, each tab uses `role="tab"`, and each panel uses `role="tabpanel"` to communicate structure to assistive technologies.
- **2.1.1 Keyboard** — Tabs are navigable with Arrow keys within the tablist; Tab moves focus into the active panel. Carousel scroll buttons are keyboard-accessible.
- **2.4.3 Focus Order** — Focus moves logically from the active tab button into the associated panel content.
- **2.4.6 Headings and Labels** — The `label` prop exposes a visible or programmatic name for the navigation landmark in the navigation variant.
- **4.1.2 Name, Role, Value** — `aria-selected`, `aria-controls`, and `aria-labelledby` associations are managed automatically by the component.
