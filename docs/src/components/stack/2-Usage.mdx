## When to Use

- Create flexible layouts with consistent spacing between elements
- Build component compositions (cards, form sections, lists)
- Manage responsive layouts without writing custom CSS
- Arrange items vertically or horizontally with automatic spacing
- Align and distribute child elements with flexbox utilities

## When NOT to Use

- For simple single-element layouts — use native HTML without wrapper
- When performance is critical and minimal DOM nesting is required
- For table-like data — use proper `<table>` elements with semantic HTML
- As a replacement for CSS Grid for complex 2D layouts
- For content that doesn't have a linear flow pattern

## Do's and Don'ts

### Spacing and Direction

Use the stack for layouts with consistent spacing. Choose direction (row/column) based on content flow.
