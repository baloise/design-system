## When to Use

- Displaying structured relational data with clearly defined rows and columns.
- Comparing multiple items across the same set of attributes.
- Listing records that users need to scan, sort, or select.

## When NOT to Use

- Avoid using a table for layout purposes — use flex or grid instead.
- Do not use a table when the data has no meaningful column relationship.
- Avoid tables for single-column data; a list is more appropriate.
