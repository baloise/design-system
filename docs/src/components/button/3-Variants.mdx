<StoryHeading of={ButtonStories.Variants}></StoryHeading>

Buttons are available in four main levels: **primary**, **secondary**, **tertiary**, and **link**. The primary level is specifically designed for prominent call-to-action elements.
Tertiary buttons can be customized with branding colors to align with the overall theme. However, themed tertiary buttons are not suitable for use on non-white backgrounds.

<CanvasTabs htmlOf={ButtonStories.VariantsHtml} of={ButtonStories.Variants} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.Sizes}></StoryHeading>

Buttons are available in two sizes: **small** and **normal** (default). Use small buttons in compact layouts such as data tables or inline toolbars. The HTML/CSS stylesheet also supports `large` and `x-large` sizes.

<CanvasTabs htmlOf={ButtonStories.SizesHtml} of={ButtonStories.Sizes} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.Inverted}></StoryHeading>

Use the `inverted` prop to display buttons on dark or colored backgrounds. Inverted variants are available for primary, secondary, and link colors.

<CanvasTabs htmlOf={ButtonStories.InvertedHtml} of={ButtonStories.Inverted} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.WithIcon}></StoryHeading>

Buttons can include a leading icon via the `icon` prop, or a trailing icon via `iconRight`. Icons should reinforce — not replace — the button label.

<CanvasTabs htmlOf={ButtonStories.WithIconHtml} of={ButtonStories.WithIcon} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.States}></StoryHeading>

Buttons support `disabled` and `loading` states. When loading, the label is replaced by a spinner to signal an async operation in progress.

<CanvasTabs htmlOf={ButtonStories.StatesHtml} of={ButtonStories.States} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.Dashed}></StoryHeading>

Dashed buttons use a dashed border and are available in the four tertiary branding colors. Pair with a circular icon for an "add" affordance.

<CanvasTabs htmlOf={ButtonStories.DashedHtml} of={ButtonStories.Dashed} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.AlertButtons}></StoryHeading>

Alert color variants — `info`, `success`, `warning`, and `danger` — are used in response or notification contexts to communicate the severity of an action.

<CanvasTabs htmlOf={ButtonStories.AlertButtonsHtml} of={ButtonStories.AlertButtons} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.SquareButtons}></StoryHeading>

Square and circle (rounded) buttons contain only an icon. Always add `aria-label` to describe the action. Use square for grid/toolbar contexts and circle for floating actions.

<CanvasTabs htmlOf={ButtonStories.SquareButtonsHtml} of={ButtonStories.SquareButtons} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.ButtonGroup}></StoryHeading>

Use the `.buttons` container to group multiple buttons horizontally. Alignment can be set with `.is-centered` or `.is-right`.

<Canvas of={ButtonStories.ButtonGroup} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.ButtonGroupAlignment}></StoryHeading>

<Canvas of={ButtonStories.ButtonGroupAlignment} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.ButtonGroupAsRow}></StoryHeading>

Use `.as-row` to place buttons in a space-between row layout — ideal for back/next navigation patterns.

<Canvas of={ButtonStories.ButtonGroupAsRow} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.ButtonGroupAsCol}></StoryHeading>

Use `.as-col` for vertical button stacks. Combine with `.is-left`, `.is-centered`, or `.is-right` for alignment.

<Canvas of={ButtonStories.ButtonGroupAsCol} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.Link}></StoryHeading>

The `.link` class styles an `<a>` element as an inline text link. Use the inverted variant on dark backgrounds.

<Canvas of={ButtonStories.Link} />

{/* ------------------------------------------------------ */}

<StoryHeading of={ButtonStories.LinkButton}></StoryHeading>

An `<a>` element styled as a button using the `.button.is-link` class. Use when the action navigates rather than submits.

<Canvas of={ButtonStories.LinkButton} />
