## CSS Shadow Parts

Shadow DOM parts are named elements within a web component that can be targeted with CSS selectors from outside the component, allowing you to style specific internal elements without breaking encapsulation.

<br />

**Example:** Target the native button element and the label:

```css
/* Style the native <button> or <a> element */
ds-button::part(native) {
  border-radius: 999px;
  letter-spacing: 0.05em;
}

/* Style the text label span */
ds-button::part(label) {
  font-weight: 700;
}

/* Style the loading spinner */
ds-button::part(spinner) {
  opacity: 0.6;
}
```

<br />

`}
</Markdown>

## Design Tokens

CSS variables (custom properties) allow you to define reusable values that can be customized globally or per component instance to override default styling without modifying the component code.

<br />

**Example:** Override button colors using CSS variables:

```css
/* Override globally for all primary buttons */
:root {
  --ds-button-color-primary-base-text: #ffffff;
  --ds-button-color-primary-base-background: #005eb8;
}

/* Override for a specific button instance */
ds-button {
  --button-color-background: #fae052;
  --button-color-text: #1a1a1a;
}
```
