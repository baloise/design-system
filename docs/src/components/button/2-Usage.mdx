## When to Use

- Trigger an action or event (form submit, navigation, opening a dialog)
- Provide a clear call-to-action with a visible label
- Group related actions in a button group or toolbar
- Show a loading state while an async operation is in progress

## When NOT to Use

- For navigation between pages — use a plain `<a>` link or the `link` color variant instead
- As a container for complex content — buttons should contain only short labels and/or an icon
- To display status information — use a badge or tag component instead
- When no interaction is available — do not render a disabled button as a placeholder

## Do's and Don'ts

### Button Labels

Labels should be short, specific, and action-oriented. Avoid generic verbs like "Click here" or "Submit".

### Icon-only Buttons

Always provide an accessible label for icon-only square or circle buttons.

  items={[
