## When to Use

- Display page titles, section headers, or any hierarchical text structure
- Provide semantic HTML heading elements for SEO and accessibility
- Create visual distinction between different heading levels in your layout
- Use shadow prop for headings displayed over images or dark backgrounds

## When NOT to Use

- Do not use headings for styling regular body text — use the Text component instead
- Do not skip heading levels (h1 to h3) — maintain semantic hierarchy
- Do not use multiple h1 elements on a single page — maintain proper document structure
- For styling only without semantic structure — consider using the Text component with visual level

## Do's and Don'ts

### Semantic Hierarchy

Maintain proper heading hierarchy in your document. Each page should have exactly one h1 element, with subsequent headings following a logical order.

<UsageExamples
  items={[
    {

          <ds-heading level="h1">Page Title</ds-heading>

      description: 'Skipping levels confuses screen readers and breaks semantic structure',
    },
  ]}

### Visual Level vs Markup Level

Use the `visualLevel` prop when you need visual styling independent of semantic markup.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Visual override when needed',
      content: (
        <>
          <Code noPreview code={'<ds-heading level="h2" visualLevel="h1">Styled as H1</ds-heading>'} />
        </>
      ),
      description: 'Allows flexible styling while maintaining semantic markup',
    },
    {
      type: 'incorrect',
      title: 'Incorrect: Wrong semantic level',
      content: (
        <>
          <Code noPreview code={'<ds-heading level="h3">Main title</ds-heading>'} />
        </>
      ),
      description: 'Using h3 as main title breaks document structure',
    },
  ]}
/>
