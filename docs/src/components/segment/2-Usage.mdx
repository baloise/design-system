## When to Use

- Let users pick exactly one option from a small set of mutually exclusive choices (2–5 items)
- Replace a group of radio buttons when the options benefit from a compact, button-like appearance
- Provide quick mode or filter switching (e.g. Monthly / Yearly, List / Grid)
- Use inside forms when the label and description context clarify the choice being made

## When NOT to Use

- When the number of options exceeds five — a select or radio group is easier to scan
- When multiple selections are allowed — use checkboxes instead
- When options are long sentences or need rich formatting — a radio group with full descriptions is clearer
- When the choice is not part of a form and has an immediate page-level effect — consider tabs or a toggle instead

## Do's and Don'ts

### Labelling the Group

Always provide a label so users know what they are selecting.

### Number of Options

Keep the option count small so the segment remains compact and scannable.
