## WCAG 2.2 AA Compliance

The steps component implements patterns that meet WCAG 2.2 Level AA standards:

- **1.3.1 Info and Relationships** — The panels variant uses `role="tablist"`, `role="tab"`, and `role="tabpanel"` with `aria-controls` and `aria-labelledby` linkage. The navigation variant uses a `<nav>` landmark with an `<ol>` list.
- **1.4.1 Use of Color** — Step states are communicated through both color changes and distinct icon/symbol content (checkmark, exclamation mark), not color alone.
- **2.1.1 Keyboard** — All steps are reachable and activatable by keyboard. Arrow keys (Left/Right or Up/Down in vertical mode), Home, and End navigate between steps in the panels variant.
- **2.4.3 Focus Order** — Focus moves to the newly selected step after keyboard activation, maintaining a logical focus sequence.
- **4.1.2 Name, Role, Value** — Each step exposes `aria-selected` and `aria-disabled` states. The active panel is visible and the inactive panels have `aria-hidden="true"`.
