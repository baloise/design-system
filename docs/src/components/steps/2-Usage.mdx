## When to Use

- Multi-step processes where the user must complete steps in order, such as checkout, onboarding, or wizards.
- When progress visibility helps the user understand how far along they are and how many steps remain.
- When navigating between major sections of a form or flow that each require distinct user input.
- As a navigation landmark (`navigation` variant) when steps represent separate pages in a linear flow.

## When NOT to Use

- For simple tabbed content with no sequential relationship — use `ds-tabs` instead.
- When there are fewer than two steps.
- When all steps are always accessible regardless of completion order — consider `ds-tabs`.
- When steps have no meaningful labels — the circle index alone is not sufficient for orientation.
