## WCAG 2.2 AA Compliance

The `ds-modal` component implements patterns that meet WCAG 2.2 Level AA standards:

- **1.3.1 Info and Relationships** — The native `<dialog>` element provides semantic role `dialog`, and `aria-labelledby` links the dialog to its heading.
- **2.1.1 Keyboard** — The modal is fully operable by keyboard: Tab cycles through focusable elements, Escape closes the dialog when `closable` is true.
- **2.1.2 No Keyboard Trap** — Focus is contained within the open modal (native browser focus trap via `showModal()`), and the user can always leave via Escape or an action button.
- **2.4.3 Focus Order** — Focus moves into the modal when it opens and returns to the triggering element when it closes.
- **4.1.2 Name, Role, Value** — The dialog role and accessible name are provided automatically through the native `<dialog>` element and `aria-labelledby`.
