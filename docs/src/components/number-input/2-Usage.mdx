## When to Use

- Collect quantity or amount inputs
- Input numeric values with specific ranges (age, quantity, price)
- Set numeric preferences with increment/decrement controls
- Ensure data validation for numeric entries
- Provide clear min/max value constraints

## When NOT to Use

- For phone numbers or codes that shouldn't be mathematically treated — use regular input
- For date/time values — use appropriate date/time inputs
- For simple yes/no choices — use checkbox or radio
- When you need extensive text entry — use regular input or textarea
- For measurements with units — consider adding unit selector separately

## Do's and Don'ts

### Min/Max Constraints

Always define min/max constraints when applicable. Users need to know the acceptable range.
