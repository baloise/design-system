## Accessibility

### HTML / CSS only

`ds-snackbar` is a web component with no CSS-only mode. All accessibility behavior is handled automatically by the component.

### Web component

The `ds-snackbar` element manages the following automatically:

- **`role="status"`** — applied to the host so screen readers announce the notification without interrupting the current reading flow
- **`aria-live="polite"`** — waits for the user to finish the current interaction before announcing; appropriate for persistent, non-urgent notifications
- **`aria-atomic="true"`** — ensures the full notification content is read as a single unit
- **Icon semantics** — the status icon (`ds-icon`) is decorative; meaning is conveyed by the heading and message text
- **Mobile collapse** — on mobile breakpoints the snackbar collapses to a square icon button (`ds-button`); clicking it expands the full notification; the button is keyboard-accessible via Tab and Enter/Space
- **Close button** — rendered as a `<ds-close>` element (a native `<button>`) when `closable` is `true`; keyboard-accessible via Tab and Enter/Space
- **Action slot** — consumers can place `<ds-button>` or `<ds-button-group>` in the `action` slot; those elements bring their own keyboard semantics
