## Accessibility

### HTML / CSS only

`ds-toast` is a web component with no CSS-only mode. All accessibility behavior is handled automatically by the component.

### Web component

The `ds-toast` element manages the following automatically:

- **`role="alert"`** — applied to the host so screen readers announce the notification immediately when it appears
- **`aria-live="assertive"`** — interrupts the current reading flow to announce the toast; appropriate for time-sensitive notifications
- **`aria-atomic="true"`** — ensures the full notification content is read as a single unit, not piecemeal
- **Icon semantics** — the status icon (`ds-icon`) is decorative and carries no additional ARIA role; meaning is conveyed by the heading and message text
- **Auto-dismiss pause** — when `duration` is set, hovering the toast pauses the timer via `mouseenter`/`mouseleave`; focus management is left to the browser
- **Close button** — rendered as a `<ds-close>` element (a native `<button>`) when `closable` is `true`; fully keyboard-accessible via Tab and Enter/Space
- **Action button** — rendered as a `<ds-button>` (native `<button>` or `<a>`) when the `action` prop is set; keyboard-accessible via Tab and Enter/Space
