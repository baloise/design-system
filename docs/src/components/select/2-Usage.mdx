## When to use

- **Multiple options**: When you need to present a list of mutually exclusive options
- **Limited space**: When you want to conserve vertical space compared to radio buttons
- **Forms**: Standard form control for selecting a single value
- **Dependent fields**: When selection determines content of other form fields
- **Required selection**: When a choice must be made before proceeding

## When NOT to use

- **Few options (2-3)**: Use radio buttons instead for better discoverability
- **Options always visible**: Use radio buttons or checkboxes if all options should be visible
- **Complex options**: If options need detailed descriptions or custom rendering, consider a different pattern
- **Single value**: For a single boolean, use a checkbox or toggle
