## When to Use

- Wrap main page content in semantic containers
- Organize content with clear sections and headers
- Provide consistent spacing and layout structure
- Create navigable content regions
- Structure complex multi-section layouts
- Implement breadcrumb or header navigation areas

## When NOT to Use

- For simple single-element layouts — basic divs may suffice
- For non-content areas (sidebars, modals) — use appropriate container components
- For content that doesn't need structural organization — use standard HTML elements
- For purely decorative grouping — unnecessary if no semantic value
- For performance-critical layouts requiring minimal markup — reduce nesting
