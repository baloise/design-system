## When to Use

- Collect text input for names, addresses, or other single-line data
- Accept email addresses with email type validation
- Accept passwords with secure masking
- Create search inputs
- Validate user input with real-time feedback
- Show required/optional field indicators

## When NOT to Use

- For multi-line text — use textarea component instead
- For selecting from predefined options — use select, radio, or checkbox
- For yes/no questions — use checkbox or toggle
- For date/time selection — use date/time inputs or picker components
- For collecting large amounts of text

## Do's and Don'ts

### Input Labels

Always pair inputs with associated labels. Never rely on placeholder text alone.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Label with input field',
      content: (
        <div>
          <label htmlFor="email-input">Email Address</label>
          <ds-input id="email-input" type="email" placeholder="your@email.com"></ds-input>
        </div>
      ),
      description: 'Label clearly identifies the field purpose',
    },
    {
      type: 'incorrect',
      title: 'Incorrect: No label, only placeholder',
      content: (
        <ds-input type="email" placeholder="Email address"></ds-input>
      ),
      description: 'Placeholder disappears when user types',
    },
  ]}
/>

### Input Validation

Show validation feedback in real-time as users type. Provide clear error messages.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Clear validation message',
      content: (
        <>
          <ds-input type="email" value="invalid" invalid></ds-input>
          <p style={{ color: '#e85c41', margin: '8px 0 0 0', fontSize: '14px' }}>Please enter a valid email address</p>
          <div style={{ textAlign: 'left', marginTop: '12px' }}>
            <Code noPreview code={'<ds-input type="email" invalid></ds-input>\n<p>Please enter a valid email address</p>'} />
          </div>
        </>
      ),
      description: 'Users know exactly what\'s wrong',
    },
    {
      type: 'incorrect',
      title: 'Incorrect: Vague error feedback',
      content: (
        <div>
          <ds-input type="email" invalid></ds-input>
          <p style={{ color: '#e85c41', margin: '8px 0 0 0' }}>Error</p>
        </div>
      ),
      description: 'Unclear what the error is',
    },
  ]}
/>
