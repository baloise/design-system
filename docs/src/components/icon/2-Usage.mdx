## When to Use

- Enhance button or link labels with visual context
- Indicate file types or status in lists
- Show required field markers in forms
- Provide visual hierarchy and break up text
- Create icon-only buttons with proper ARIA labels
- Display status or state indicators

## When NOT to Use

- As the only way to convey meaning — always pair with text for critical information
- For decorative purposes at high scale — use small, subtle icons for decoration
- Without proper labels on interactive icon-only elements
- For essential information that users might not understand at a glance
- At sizes too small to be recognizable or clickable

## Do's and Don'ts

### Icon + Text Pairing

Icons should reinforce, not replace, text labels. Always provide accompanying text for meaningful content.

### Icon Accessibility

Always provide meaningful labels for icon-only interactive elements.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Accessible icon button',
      content: (
        <>
          <ds-button square aria-label="Download file">
            <ds-icon name="download"></ds-icon>
          </ds-button>
          <div style={{ textAlign: 'left', marginTop: '12px' }}>
            <Code noPreview code={'<ds-button square aria-label="Download file">\n  <ds-icon name="download"></ds-icon>\n</ds-button>'} />
          </div>
        </>
      ),
      description: 'Screen readers announce the button purpose',
    },
    {
      type: 'incorrect',
      title: 'Incorrect: No accessible label',
      content: (
        <ds-button square>
          <ds-icon name="download"></ds-icon>
        </ds-button>
      ),
      description: 'Screen reader users won\'t know the button purpose',
    },
  ]}
