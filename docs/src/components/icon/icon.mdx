## UI Icons

A minimal set of UI-Icons is already available from the beginning. Here are some [more UI-Icons](?path=/docs/foundation-icons--documentation#overview), which can be imported

<Canvas of={IconStories.UiIcons} sourceState="none" />

To incorporate icons beyond this basic set, you can either **import them during initialization** or **import the raw SVG icon** and pass it to the component using the `svg` property.

<AngularFramework>

### Import during initialization

Import the icons form the `@baloise/ds-icons` package and pass them with the BalConfig during the initialization.

```ts
import { provideDesignSystem } from '@baloise/ds-angular'
import { IconStarFull } from '@baloise/ds-assets'

export const appConfig: ApplicationConfig = {
  providers: [
    provideDesignSystem({
      defaults: {
        icons: { IconStarFull },
      },
    }),
  ],
}
```

### Import into the component

When importing an SVG into your component you decrease the initial page load. So you use the SVG only where you needed and at the time you wanted.

```ts
import { IconStarFull } from '@baloise/ds-assets'

@Component({...})
export class AppComponent {
  myIcon = IconStarFull
}

```

```html
<ds-icon [svg]="myIcon"></ds-icon>
```

</AngularFramework>

<ReactFramework>

### Import during initialization

Import the icons form the `@baloise/ds-assets` package and pass them with the BalConfig during the initialization.

```ts
import React from 'react'
import { useDesignSystem, App } from '@baloise/ds-react'
import { IconStarFull } from '@baloise/ds-assets'

export default function App() {
  useDesignSystem({
    defaults: {
      icons: { IconStarFull },
    },
  })

  return <BalApp>...</BalApp>
}
```

### Import into the component

When importing an SVG into your component you decrease the initial page load. So you use the SVG only where you needed and at the time you wanted.

```ts
import React from 'react'
import { Icon } from '@baloise/ds-react'
import { IconStarFull } from '@baloise/ds-assets'

export default function Example() {
  return <BalIcon svg={IconStarFull}></BalIcon>
}
```

</ReactFramework>

<HTMLFramework>

### Import during initialization

Import the icons form the `@baloise/ds-assets` package and pass them with the BalConfig during the initialization.

```ts
import { initialize } from '@baloise/ds-core'
import { IconStarFull } from '@baloise/ds-assets'

initialize({
  icons: { IconStarFull },
})
```

### Import into the component

When importing an SVG into your component you decrease the initial page load. So you use the SVG only where you needed and at the time you wanted.

```js
import { IconStarFull } from '@baloise/ds-assets'

document.getElementById('my-icon').svg = IconStarFull
```

</HTMLFramework>

## Accessibility

### HTML / CSS only

Icons are **purely decorative** by default. If an icon conveys meaning (e.g., in a button or link), add an `aria-label` to its parent control, not the icon itself.

<Markdown>
  {`
| Requirement        | How to implement                                                                               |
| ------------------ | ------------------------------------ |
| Decorative icons   | Icons in buttons/links are automatically hidden from screen readers via \`aria-hidden="true"\` |
| Icon in button     | Add \`aria-label\` to the \`<button>\` or \`<ds-button>\`, not the icon                       |
| Icon with text     | If icon has adjacent text label, the text is sufficient; icon remains \`aria-hidden\`         |
| Meaningful icon    | In rare cases where icon alone conveys meaning, add \`role="img"\` and \`aria-label\` to icon |
| Color alone        | Do not use color alone to convey meaning; pair with text or icon semantics                   |
`}
</Markdown>

> **Note:** By default, `ds-icon` sets `aria-hidden="true"` on the host element. This is correct for decorative use. If you need an icon to be announced, wrap it in a labeled button or link instead.

### Web component

The `ds-icon` element is a **decorative presentation component**:

- **`aria-hidden="true"`** — automatically set on the host; icons are not announced by screen readers by default
- **No native interactivity** — icons are non-interactive by default; if clickable, wrap in `<ds-button>` or `<a>` with a label
- **SVG sanitization** — custom SVG content is sanitized for security; inline SVGs are embedded safely
- **Color parsing** — the `color` prop affects visual styling only; does not convey meaning to AT
- **Disabled state** — the `disabled` prop sets the color to grey visually; does not affect interactivity (icon is not interactive by default)
