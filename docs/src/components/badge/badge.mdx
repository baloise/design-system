## Accessibility

### HTML / CSS only

Badges are **presentational status indicators** (counts, flags, notifications). Always include descriptive text content or an `aria-label` on the parent element.

<Markdown>
  {`
| Requirement           | How to implement                                                                               |
| --------------------- | -------- |
| Badge content         | Always include text (count, abbreviation, etc.) or wrap badge in labeled container             |
| Meaningful content    | Avoid badges with only icons; if icon-only, add \`aria-label\` to parent                     |
| Color semantics       | Do not rely on color alone (e.g., red for error); pair with text or semantic context          |
| Decorative badge      | If purely decorative count, add \`aria-hidden="true"\` to badge                               |
| Positioned badge      | When positioned on a button/card, badge text helps users understand the status                |
`}
</Markdown>

> **Note:** Badges convey meaning through content (usually text or number), not appearance alone. If your badge is icon-only with position styling, ensure the parent element (button, tab, card) has an accessible label that explains what the badge indicates.

### Web component

The `ds-badge` element is a **presentational status indicator**:

- **No ARIA by default** — badges rely on text content for meaning; add labels to parent if needed
- **Slot for content** — any slotted text (numbers, abbreviations, icons) is announced by AT
- **Icon support** — the `icon` prop adds an icon via `ds-icon` with `aria-hidden="true"`; only text content is announced
- **Color & size variants** — affect styling only; do not convey semantic meaning on their own
- **Pulse animation** — visual feedback for new notifications; does not affect AT
