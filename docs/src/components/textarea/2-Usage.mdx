## When to Use

- Collect user feedback or comments
- Accept longer text descriptions
- Create text editors for note-taking
- Collect detailed messages or reports
- Allow multi-line input for addresses or special instructions

## When NOT to Use

- For single-line text — use input component instead
- For predefined options — use select, radio, or checkbox
- For very short responses — use input field
- When space is extremely limited — consider collapsible or modal form
- For code or structured data — use appropriate input types

## Do's and Don'ts

### Textarea Labels and Instructions

Always provide clear labels and help text. Show character limits if applicable.

<UsageExamples
  items={[
    {
      type: 'correct',
      title: 'Correct: Label with instructions',

      description: 'Clear guidance on what to enter',

    {

### Character Limits and Counters

Show character counts for textareas with limits. Help users understand remaining space.

<UsageExamples
  items={[
    {
      type: 'correct',
