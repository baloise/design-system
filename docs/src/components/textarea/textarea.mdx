## Accessibility

### HTML / CSS only

Textareas are **form control elements** for multi-line text entry. Always provide a visible label and optional helper text. Link textareas to labels and descriptions using standard HTML attributes.

<Markdown>
  {`
| Requirement            | How to implement                                                                                             |
| ---------------------- | ------------------------------------ |
| Label association      | Wrap \`<textarea>\` in \`<label for="textarea-id">\` or use \`<label><textarea></textarea></label>\`       |
| Textarea ID            | Set a unique \`id\` on each \`<textarea>\` so labels and descriptions can reference it                     |
| Description text       | Link helper/error text via \`aria-describedby="help-id"\` on the textarea element                          |
| Required state         | Set \`required\` attribute on the textarea; label should show required indicator                            |
| Invalid state          | Set \`aria-invalid="true"\` on textarea when there are validation errors                                    |
| Disabled state         | Set \`disabled\` attribute on textarea; visually distinct styling via CSS                                   |
| Readonly state         | Set \`readonly\` attribute when textarea should not be edited but is still submitted                        |
| Min/Max length         | Set \`minlength\` and \`maxlength\` attributes to enforce input constraints; announce limits to users       |
| Resize control         | Use CSS \`resize: none\` sparingly; avoid hiding user's ability to resize unless there is strong UX reason  |
| Placeholder not label  | Use \`placeholder\` only for hints; never rely on it as the primary label                                   |
`}
</Markdown>

### Web component

The `ds-textarea` element manages the following automatically:

- **Label linking** — the Field component renders a `<label>` with `htmlFor` pointing to the textarea's ID
- **`aria-describedby`** — automatically set on the native `<textarea>` to link to the description element ID
- **`aria-invalid`** — set to `"true"` when `invalid` prop is true; `"false"` otherwise
- **`aria-required`** — accessible via the `required` prop forwarded to the native textarea (defaults to `true`)
- **`disabled` attribute** — forwarded from the component prop to the native textarea
- **`readonly` attribute** — forwarded from the component prop to the native textarea
- **Form association** — uses `ElementInternals` to participate in native `<form>` submit/reset/validity without wrapper elements
- **Length constraints** — `minLength` and `maxLength` props forwarded to native textarea for built-in validation
- **Dimensions** — `rows` and `cols` props control visible size; `wrap` prop controls line-wrapping behavior
- **Dynamic feedback** — `invalidText` prop displays error messages linked via the description element
