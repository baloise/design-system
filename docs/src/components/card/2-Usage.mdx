## When to Use

- Display related content as a self-contained module
- Create product, article, or profile summary cards
- Group form fields or steps in a focused container
- Highlight featured content or promotions
- Build grids of related items (gallery, product listing)

## When NOT to Use

- For simple content that doesn't require visual separation — use plain dividers or spacing instead
- As the only layout method on a page — combine cards with other layout components
- To hide content that should be immediately visible — do not nest important information inside cards
- For very simple elements like single buttons or links — cards add unnecessary overhead
