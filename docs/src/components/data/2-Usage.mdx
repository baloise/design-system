## When to Use

- Displaying read-only label-value pairs (user profile information, contact details)
- Showing structured data in a consistent, accessible format
- Creating data entry forms with custom input controls via slots
- Presenting hierarchical or nested information with clear labels

## When NOT to Use

- For simple key-value displays without visual hierarchy — use plain text or lists instead
- For complex tabular data with multiple columns and rows — use a table component instead
- For form layouts with validation and error handling — use a form component instead
- For displaying unstructured or narrative content
