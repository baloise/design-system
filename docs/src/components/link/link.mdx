## Accessibility

### HTML / CSS only

Use semantic HTML `<a>` elements for navigation links. The `ds-link` component provides styling and visual variants.

<Markdown>
  {`
| Requirement           | How to implement                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------- |
| Link href             | Always set the \`href\` attribute; screen readers announce the link and keyboard users can activate it        |
| Link text / label     | Provide descriptive link text visible in the \`<a>\` element; avoid generic text like "click here"            |
| Icon-only links       | If a link contains only an icon with no visible text, add \`aria-label\` to the \`<a>\` element              |
| Icon accessibility    | Set \`aria-hidden="true"\` on decorative \`<ds-icon>\` elements inside the link text                         |
| External link         | Add \`target="_blank"\` if opening in a new tab; consider adding \`aria-label\` to warn users (optional)      |
| Disabled appearance   | Do not use CSS class \`is-disabled\` on \`<a>\` elements; links cannot be disabled semantically               |
`}
</Markdown>

> **Note:** The `is-disabled` CSS class in the style examples is visual only and does not prevent keyboard or programmatic activation. If you need a disabled link, use a `<button>` instead or add `aria-disabled="true"` and handle clicks with JavaScript.

### Web component

The `ds-link` component is a CSS-only presentational layer:

- **No custom elements** — renders standard HTML `<a>` elements; all accessibility comes from native HTML semantics
- **No ARIA attributes** — the component adds no ARIA; developers must provide `href`, link text, and `aria-label` if needed
- **No state management** — the component provides visual styling only; disabled state must be handled with `aria-disabled` + JavaScript if required
- **Keyboard navigation** — native; the `<a>` element is automatically keyboard-focusable and activatable via Enter key
- **Screen reader support** — relies on descriptive link text and the `href` attribute; icon-only links need `aria-label`
