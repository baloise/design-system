## When to Use

- Display brief, temporary notifications that auto-dismiss after a short duration
- Show confirmation messages for completed actions (e.g., "File saved successfully")
- Provide non-blocking feedback that doesn't require user action
- Inform users of background processes or updates that don't demand immediate attention

## When NOT to Use

- For notifications that need to remain visible for extended periods — use **Snackbar** instead
- For critical errors or warnings that require user interaction — use **Alert** in a modal dialog
- For large amounts of content or complex information — use a dedicated notification panel
- For form validation errors — use inline field-level feedback
