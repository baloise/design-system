## Accessibility

### HTML / CSS only

Since **pagination is a web component** (`ds-pagination`), it handles all accessibility requirements automatically through the Shadow DOM. There is no CSS-only mode for this component.

### Web component

The `ds-pagination` element manages the following automatically:

- **Navigation landmark** — renders a `<nav>` element with `role="navigation"` and `aria-label` to identify the pagination section for screen readers
- **Current page indicator** — sets `aria-current="true"` on the active page button so screen readers announce which page is currently selected
- **Button labels** — each button has an `aria-label` and `title` attribute describing its function (e.g. "Page 2", "Previous page", "Next page")
- **Internationalized labels** — all button labels are translated based on the component's `language` configuration (DE, EN, FR, IT, NL, ES, PL, PT, SV, FI)
- **Disabled state** — when `disabled="true"`, all page buttons are set to `disabled` and previous/next buttons are disabled based on current page
- **Decorative elements** — the ellipsis (`…`) is marked with `aria-hidden="true"` since it is purely visual
- **Keyboard interaction** — all buttons are keyboard accessible; users can tab through and press Enter/Space to select pages
- **Icon accessibility** — navigation icons (`nav-go-left`, `nav-go-right`) inherit the disabled state from their parent buttons
