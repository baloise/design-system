## When to Use

- Navigate large lists or search results across multiple pages
- Break content into manageable chunks for readability
- Show results count and page position to users
- Provide next/previous navigation between sequential pages
- Display paginated data in tables or galleries

## When NOT to Use

- For very small datasets (fewer than 20 items) — consider showing all items
- For infinite scroll experiences — pagination breaks the paradigm
- For sequential step-by-step workflows — use step indicators or breadcrumbs
- For single long pages — pagination not needed
- For mobile with limited space — consider infinite scroll or "Load More" button
