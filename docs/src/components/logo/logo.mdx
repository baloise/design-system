## Accessibility

### HTML / CSS only

Logos are **decorative brand images**. If a logo links to the homepage, wrap it in an `<a>` tag with an `aria-label` like "Helvetia Home".

<Markdown>
  {`
| Requirement      | How to implement                                                                              |
| ---------------- | ------------------------------------ |
| Decorative logo  | If not linked, logo has \`aria-hidden="true"\` and is not announced                          |
| Logo as link     | Wrap \`<ds-logo>\` in \`<a href="/">\\</a>\` with \`aria-label="Helvetia Home"\`            |
| Animation        | Animated logos are safe; animation does not interfere with AT                                |
| Alt text         | Logo-in-header pattern: use \`<h1><a aria-label="Brand">...</a></h1>\` for document heading |
`}
</Markdown>

> **Note:** By default, logos are hidden from screen readers. If your logo is a link to the home page, provide a label via the parent link element or a nearby skip link.

### Web component

The `ds-logo` element is a **decorative brand presentation component**:

- **`aria-hidden="true"`** — automatically set unless the logo is wrapped in an interactive element
- **No interactivity** — logos are visual only; to make clickable, wrap in `<a>` with `aria-label`
- **Animation support** — the `animated` prop enables Lottie-based animation; respects system `prefers-reduced-motion`
- **Brand variants** — `brand` prop switches between Helvetia and Baloise visuals; does not affect AT
- **Size & color** — affect styling only; no semantic impact
