## When to Use

- Collecting user input across multiple fields and sections
- Building multi-step wizards or complex forms
- Creating responsive layouts that adapt across device sizes
- Organizing related form fields with consistent styling and spacing
- Building accessible forms with proper field grouping and labels
- Creating data entry interfaces with mixed field widths and alignments

## When NOT to Use

- For single input fields — use individual input, textarea, or select components
- For simple yes/no questions — use a standalone checkbox or toggle
- When a modal or dialog is more appropriate for the context
- For display-only content — use table or card components instead

## Do's and Don'ts

### Organize Fields Logically

Group related form fields together and arrange them in a logical reading order that matches the data collection flow.
