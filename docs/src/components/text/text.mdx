## Accessibility

### HTML / CSS only

Use semantic HTML to display text. The `ds-text` component is a presentation layer; its accessibility depends entirely on using appropriate HTML elements for the content's meaning.

<Markdown>
  {`
| Requirement            | How to implement                                                                                   |
| ---------------------- | -------------------------------------------------------------------------------------------------- |
| Semantic text elements | Use \`<p>\` for paragraphs, \`<span>\` for inline text, \`<strong>\` for emphasis, \`<b>\` for bold |
| Heading hierarchy      | Use \`<h1>\`–\`<h6>\` for headings; do not rely on the \`heading\` prop alone for semantics        |
| Text alternatives      | Ensure all text content is human-readable; if text is decorative or purely stylistic, mark it with \`aria-hidden="true"\` |
| Color contrast         | All text must meet WCAG 2.2 AA contrast ratios (4.5:1 for normal text, 3:1 for large text)         |
`}
</Markdown>

> **Note:** The `ds-text` web component is intended for styled text display in contexts where you already have semantic markup. When possible, prefer standard HTML elements (`<p>`, `<strong>`, `<em>`) over the web component for better performance and simpler accessibility.

### Web component

The `ds-text` element is a presentational wrapper with no interactive elements or state-driven ARIA attributes:

- **No roles** — the component renders semantic HTML elements as-is; does not alter accessibility tree
- **No state attributes** — the component has no interactive behavior (no `aria-expanded`, `aria-checked`, etc.)
- **Color is informational only** — the `color` prop applies visual styling; meaning is conveyed by text content, not color alone
- **Disabled state is visual** — the `disabled` prop applies styling only; it does not prevent interaction or participation in forms
- **Keyboard navigation** — text content is inherently selectable and copyable by keyboard and mouse
- **Screen reader support** — semantic HTML elements inside the component are announced correctly by assistive technologies
