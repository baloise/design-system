## What are CSS Utilities?

CSS utility classes provide a **utility-first** approach to styling. Instead of writing custom CSS for every component, you apply pre-defined classes directly to your HTML elements. These classes are generated using [UnoCSS](https://unocss.dev) and are based on the [design tokens](?path=/docs/foundation-design-tokens--documentation), ensuring consistent design across your application.

<CanvasWithCodePen of={BackgroundColorStories.Basic} />

This example demonstrates:
- **Spacing**: `p-base` (padding)
- **Background**: `bg-green` (background color)

### Why use CSS Utilities?

- **Speed**: Build layouts and components faster without writing CSS
- **Consistency**: All utilities are tied to design tokens, guaranteeing design system compliance
- **Maintainability**: Changes to design tokens automatically reflect in utility classes
- **Flexibility**: Combine utilities to create any design without breaking components
- **Responsive**: Built-in support for responsive design at multiple breakpoints
- **Performance**: Generate only the CSS you actually use with UnoCSS

## Installation

The `@baloise/ds-css` package is included when you install `@baloise/ds-core`, so if that is already installed skip the next step.

```bash
npm add @baloise/ds-css
```

## Usage

### CSS Only (Web Components)

Import the compiled CSS directly into your HTML or bundler:

```html
<!-- Full design system (base + components + utilities) -->
<link rel="stylesheet" href="@baloise/ds-css/dist/css/design-system.css" />

<!-- Or import selectively -->
<link rel="stylesheet" href="@baloise/ds-css/dist/css/base.css" />
<link rel="stylesheet" href="@baloise/ds-css/dist/css/components.css" />
<link rel="stylesheet" href="@baloise/ds-css/dist/css/utilities.css" />
```

### SCSS/Sass (Monorepo Development)

Use `@use` to import Sass utilities and design tokens:

```scss
// Design tokens and Sass functions
@use '@baloise/ds-css/dist/scss/design-system';

// Use CSS custom properties in your styles
.my-component {
  background: var(--ds-alias-background-color-primary);
  padding: var(--ds-alias-space-base-device);
}
```

## Using Utilities in Angular

Utility classes can be applied directly to template elements:

```typescript
// my-component.component.ts
import { Component } from '@angular/core'

@Component({
  selector: 'app-my-component',
  template: `
    <div class="flex gap-md items-center justify-between px-md py-lg">
      <h1 class="text-lg font-bold text-primary">Hello</h1>
      <button class="px-md py-sm bg-primary text-white rounded hover:bg-primary-hover">
        Click me
      </button>
    </div>
  `,
})
export class MyComponent {}
```

### Optimizing Output with UnoCSS

To generate **only the utility classes you actually use** (reducing CSS file size), configure UnoCSS in your build process:

#### 1. Install UnoCSS

```bash
npm add -D unocss
```

#### 2. Configure UnoCSS (in your bundler or build script)

For **Angular** with `@angular-eslint/schematics`, add UnoCSS to your Webpack/esbuild config:

```typescript
// vite.config.ts or angular.json (if using Vite)
import { defineConfig } from 'vite'
import UnoCSS from 'unocss/vite'
import { presetDsUtilities } from '@baloise/ds-css/preset'

export default defineConfig({
  plugins: [
    UnoCSS({
      presets: [presetDsUtilities()],
      // Scan these file patterns for class names
      content: {
        filesystem: ['src/**/*.{ts,tsx,html}'],
      },
    }),
  ],
})
```

#### 3. Extract Used Classes Only

UnoCSS will scan your templates and only generate CSS for classes that are actually present:

```html
<!-- Input template -->
<div class="flex gap-md px-lg py-md">Content</div>
<button class="px-md py-sm bg-primary">Button</button>

<!-- Output: Only these classes are in the final CSS file -->
.flex { display: flex; }
.gap-md { gap: var(--ds-spacing-md); }
.px-lg { padding-left: var(--ds-spacing-lg); padding-right: var(--ds-spacing-lg); }
.py-md { padding-top: var(--ds-spacing-md); padding-bottom: var(--ds-spacing-md); }
.px-md { padding-left: var(--ds-spacing-md); padding-right: var(--ds-spacing-md); }
.py-sm { padding-top: var(--ds-spacing-sm); padding-bottom: var(--ds-spacing-sm); }
.bg-primary { background: var(--ds-color-primary); }
```

#### 4. Enable in Your Angular Application

For a full Angular app setup, import the generated CSS in your `main.ts` or global styles:

```typescript
// main.ts
import 'uno.css' // Auto-generated by UnoCSS
import '@baloise/ds-css/dist/css/base.css'
import '@baloise/ds-css/dist/css/components.css'

bootstrapApplication(AppComponent)
```

This approach ensures your final CSS bundle contains **only the utilities you use**, keeping your application lightweight.
