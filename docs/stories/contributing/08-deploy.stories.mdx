The Design System documentation is deployed on Vercel automatically.
Vercel is listening on Git changes and then triggers new deployments.

## Domains

<Markdown>
  {`
| Domains                                                                                  | Description                                                                              |
| ---------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| [design.baloise.dev](design.baloise.dev)                                                 | The production domain for the latest released version. It is based on the latest branch. |
| [baloise-design-preview.vercel.app](baloise-design-preview.vercel.app)                   | This points to the unreleased main branch                                                |
| [design-system-fhlyaiq8z-baloise.vercel.app](design-system-fhlyaiq8z-baloise.vercel.app) | Each pull request gets deployed with its own url                                         |
  `}
</Markdown>

## Pull Request Deployments

In the pull request Vercel adds a comment with details to the deployment.
