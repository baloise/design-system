# Rename Primitive Tokens to Global Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the primitive design token tier from "Primitive" to "Global", changing all generated CSS variable names from `--ds-color-*` / `--ds-font-*` / `--ds-size-*` / `--ds-elevation-*` to `--ds-global-color-*` / `--ds-global-font-*` / `--ds-global-size-*` / `--ds-global-elevation-*`.

**Architecture:** A single change to the style-dictionary name transformer in `packages/tokens/src/transformers.ts` drives the entire rename in generated dist files. Alias tokens that reference primitives via `var(--ds-color-*)` automatically update when tokens are rebuilt (due to `outputReferences: true`). The remaining work is find-replacing the old names in the 25 SCSS source files that reference primitive tokens directly.

**Tech Stack:** Style Dictionary (token build), SCSS, Stencil web components, Nx monorepo (`npm run tokens` rebuilds tokens).

> **⚠️ Breaking change:** This renames public CSS custom properties. Consumers using `--ds-color-*`, `--ds-font-*`, `--ds-size-*`, or `--ds-elevation-*` directly will need to update. A `major` changeset entry is required.

---

## Token Name Mapping

| Old name pattern | New name pattern |
|---|---|
| `--ds-color-danger-4` | `--ds-global-color-danger-4` |
| `--ds-font-family-heading` | `--ds-global-font-family-heading` |
| `--ds-size-space-16` | `--ds-global-size-space-16` |
| `--ds-elevation-shadow-2` | `--ds-global-elevation-shadow-2` |

Alias tokens (`--ds-background-color-*`, `--ds-text-color-*`, etc.) and component tokens (`--ds-badge-*`, `--ds-button-*`, etc.) are **not renamed** — only the four primitive categories above.

```typescript
// Before (lines ~57-65):

// After:
```
