# Design: Migrate `bal-number-input` → `ds-number-input`

**Date:** 2026-04-16
**Scope:** Migrate `bal-number-input` to the DS standard established by `ds-input` and `ds-textarea`. Approach A (Partial FormControl) is used — `FormControl` handles plumbing, custom event handlers manage numeric formatting.
**Approach:** Component-first → Styles → Utils → Tests → Docs (each checkpoint is a working build).

---

## 2. Component (`number-input.tsx`)

### Decorator

```ts
@Component({
  tag: 'ds-number-input',
  styleUrl: 'number-input.host.scss',
  shadow: true,
  formAssociated: true,
})
export class NumberInput
  implements ComponentInterface, FormControlInterface<number | null, number | undefined>, Loggable
```

### FormControl Usage — Approach A (Partial)

`FormControl` is used only for plumbing. All numeric event handling is written custom.

| Uses `FormControl` | Custom (written in component) |
|---|---|
| `control.nativeEl` — native `<input>` ref | `handleInput` — validate key, update `lastValue`, emit `dsInput` with numeric value |
| `control.setFocus()` / `control.setBlur()` | `handleFocus` — restore raw value, select all text via timeout |
| `control.connectedCallback()` | `handleBlur` — apply locale formatting, call `control.setValue(numericValue)` |
| `control.componentDidLoad()` | `handleKeydown` — call `validateKeyDown` from utils |
| `control.listenOnClick(ev)` | `handleValueChange` — `@Watch('value')` for external prop updates |
| `control.listenOnReset(ev)` | |

`control.setValue(numericValue)` (from `FormControl`) is called inside `handleBlur` — it emits `dsChange` and updates `internals.setFormValue()`.

### Value Type

| Old | New |
|---|---|
| `@Prop({ mutable: true }) value?: number \| string = undefined` | `@Prop({ mutable: true, reflect: true }) value: number \| null = null` |
| emits `number \| undefined` | emits `number \| undefined` (undefined = "no value" signal for events) |

### Props

**Kept (numeric-specific):**
`decimal`, `onlyPositive`, `exactNumber`, `suffix`, `pattern`, `min`, `max`, `placeholder`, `disabled`, `readonly`, `required`, `debounce`, `name`

**New (DS standard):**
`label`, `description`, `color: 'primary' | 'danger' | 'success' | 'warning'`, `invalidText`

**Dropped:**
`autoInvalidOff` — handled internally by `FormControl`

### Internal State

These drive the display/format separation and are not exposed as props:

- `lastValue: string` — raw numeric string without formatting (e.g. `"1234.56"`)
- `nativeInputValue: string` — what the native `<input>` actually displays (formatted on blur, raw on focus)
- `selectTimeout?: NodeJS.Timeout` — selects all text after focus

### Events

```ts
@Event() dsInput!: EventEmitter<DS.NumberInputInputDetail>     // number | undefined
@Event() dsChange!: EventEmitter<DS.NumberInputChangeDetail>   // number | undefined
@Event() dsBlur!: EventEmitter<DS.NumberInputBlurDetail>       // FocusEvent
@Event() dsFocus!: EventEmitter<DS.NumberInputFocusDetail>     // FocusEvent
@Event() dsClick!: EventEmitter<DS.NumberInputClickDetail>     // MouseEvent (required by FormControlInterface)
@Event() dsKeyPress!: EventEmitter<DS.NumberInputKeyPressDetail> // KeyboardEvent
```

### Render Structure

```tsx
<Host
  aria-disabled={ariaBooleanToString(this.disabled)}
  class={{
    'ds-field': true,
    'is-disabled': this.disabled,
    'is-danger': this.color === 'danger' || this.invalid,
    'is-success': this.color === 'success' && !this.invalid,
    'is-warning': this.color === 'warning' && !this.invalid,
  }}
>
  <label htmlFor="input" part="label" id="label">
    <slot name="label">{this.label}</slot>
    {this.required === false && <span>{I18nDsLabel[this.language].optional || ''}</span>}
  </label>

  <div id="container">
    <input
      id="input"
      part="input"
      type="text"
      inputMode="decimal"
      pattern={this.inputPattern}
      ref={el => (this.control.nativeEl = el)}
      aria-describedby="description"
      aria-invalid={this.invalid === true ? 'true' : 'false'}
      disabled={this.disabled}
      placeholder={this.placeholder || ''}
      readonly={this.readonly}
      required={this.required}
      min={this.min}
      max={this.max}
      name={this.name}
      value={this.nativeInputValue}
      onInput={ev => this.handleInput(ev)}
      onFocus={ev => this.handleFocus(ev)}
      onBlur={ev => this.handleBlur(ev)}
      onClick={ev => this.control.onClick(ev)}
      onKeyDown={ev => this.handleKeydown(ev)}
      onKeyPress={ev => this.dsKeyPress.emit(ev)}
      {...this.inheritedAttributes}
    />
  </div>

  <span id="description" part="description">
    {this.invalid && this.invalidText && <ds-icon name="alert"></ds-icon>}
    <slot name="description">{this.invalid && this.invalidText ? this.invalidText : this.description}</slot>
  </span>
</Host>
```

## 3. SCSS Architecture

No `number-input.mixin.scss` — imports `input.mixin.scss` directly.

### `number-input.host.scss`

```scss
@use '@baloise/ds-styles/sass/mixins' as *;
@use '../../vars' as vars;
@use '../input/input.mixin' as input;

:host {
  @include input.vars('input');
  @include input.vars-container('input');
  @include input.vars-label('input');
  @include input.vars-description('input');
  display: grid; gap: 0.25rem; position: relative;
  @include input.style('input', '#input');
  @include input.container('input');
  @include input.label('input');
  @include input.description('input');
}

:host(.is-danger)   { /* same --mod-input-* vars as input.host.scss */ }
:host(.is-success)  { ... }
:host(.is-warning)  { ... }
:host(.is-disabled) {
  &, #label, #description, #input { pointer-events: none; cursor: default; }
  /* disabled --mod-input-* vars */
}
```

### `number-input.style.scss`

Same structure as `input.style.scss` — targets `.field` / `.input` / `.label` / `.help` class selectors. State variants: `.field.is-danger`, `.field.is-disabled`, etc. Includes `.field + .field` spacing rule.

CSS variable prefix stays `input` throughout — textarea and number-input reuse the same `--ds-input-*` design token space.

## 5. Tests & Page Object

### `number-input.po.ts`

```ts
export class DsNumberInput extends PageObject {
  readonly nativeInput: Locator
  constructor(el: E2ELocator) { super(el); this.nativeInput = el.locator('[part="input"]') }
  async fill(value: string) { await this.nativeInput.fill(value) }
  async blur() { await this.nativeInput.blur() }
  async assertValue(value: string) { await expect(this.nativeInput).toHaveValue(value) }
  async assertToBeDisabled() { await expect(this.nativeInput).toBeDisabled() }
}
```

### `number-input.component.play.ts`

Covers numeric-specific behavior. `page.mount()` inside every test — never in `beforeEach`.

- `dsInput` fires with `number` on keystroke
- `dsChange` fires with `number` on blur
- No `dsChange` when value unchanged on blur
- Focus: shows raw number (no thousand separators)
- Blur: shows locale-formatted string with thousand separators
- `decimal="2"`: allows decimal entry
- `decimal="0"`: blocks decimal separator key
- `onlyPositive`: blocks negative sign
- `suffix="CHF"`: appended on blur, stripped on focus
- `exactNumber`: displays `0` when empty
- Disabled state
- Form reset

### `number-input.a11y.play.ts`

Mounts `ds-number-input` for each state and calls `a11y('ds-number-input')`:
`default`, `with placeholder`, `with value`, `disabled`, `invalid`, `success`, `warning`
