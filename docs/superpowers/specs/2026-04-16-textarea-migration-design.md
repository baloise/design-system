# Design: Migrate `bal-textarea` → `ds-textarea`

**Date:** 2026-04-16  
**Scope:** Two deliverables — (1) a reusable Claude skill for migrating `bal-*` components to the DS standard, and (2) the concrete migration of `bal-textarea` to `ds-textarea` as the canonical reference.  
**Approach:** Component-first → Styles → Tests → Docs (each checkpoint is a working build).

---

## 1. File Structure

The old `packages/core/src/components/bal-textarea/` directory is deleted after migration. The new layout:

```
packages/core/src/components/textarea/
  textarea.tsx
  textarea.interfaces.ts
  textarea.mixin.scss          ← new: textarea-specific container mixin, delegates rest to input.mixin
  textarea.host.scss           ← new: shadow DOM :host styles

    textarea.visual.html       ← rewritten: ds-textarea tags, new sections

```

## 2. Component (`textarea.tsx`)

### Decorator

```ts
@Component({
  tag: 'ds-textarea',
  styleUrl: 'textarea.host.scss',
  shadow: true,
  formAssociated: true,
})
export class Textarea implements ComponentInterface, FormControlInterface<string | null>, Loggable
```

### What is removed

| Removed | Replaced by |
|---|---|
| `BEM` import + `.class()` calls | Plain `id` attributes on elements |
| `BalAriaForm` / `setAriaForm` / `BalAriaFormLinking` | `FormControl` handles ARIA internally |
| `inputHandleBlur/Click/Focus/Reset/...` util functions | `FormControl` methods |
| `clickable` prop | Not part of DS standard |
| `FormInput` interface | `FormControlInterface<string \| null>` |

### New props (matching `ds-input`)

| Prop | Type | Default | Purpose |
|---|---|---|---|
| `label` | `string \| undefined` | — | Rendered inside shadow `<label>` |
| `description` | `string \| undefined` | — | Rendered in `<span id="description">` |
| `color` | `'primary' \| 'danger' \| 'success' \| 'warning'` | `'primary'` | Drives host modifier class |
| `invalidText` | `string \| undefined` | — | Shown in description when `invalid` is true |

### Kept props (textarea-specific)

`name`, `invalid`, `autocapitalize`, `autocomplete`, `autofocus`, `debounce`, `placeholder`, `maxLength`, `minLength`, `disabled`, `readonly`, `cols`, `rows`, `wrap`, `required`, `inputmode`, `value`

### Events (renamed to `ds` prefix)

`dsChange`, `dsInput`, `dsBlur`, `dsKeyPress`, `dsFocus`

### Interfaces namespace

`BalProps` / `BalEvents` namespaces become the `DS` namespace, consistent with `input.interfaces.ts`. Textarea-specific types kept: `TextareaWrap`, `TextareaInputMode`.

### Render structure

Elements use `id` attributes (not BEM class names). Host carries modifier classes.

```tsx
<Host
  aria-disabled={ariaBooleanToString(this.disabled)}
  class={{
    'ds-field': true,
    'is-disabled': this.disabled,
    'is-danger': this.color === 'danger' || this.invalid,
    'is-success': this.color === 'success' && !this.invalid,
    'is-warning': this.color === 'warning' && !this.invalid,
  }}
>
  <label htmlFor="textarea" part="label" id="label">
    <slot name="label">{this.label}</slot>
  </label>

  <div id="container">
    <textarea
      id="textarea"
      part="textarea"
      aria-describedby="description"
      aria-invalid={this.invalid === true ? 'true' : 'false'}
      ...native textarea props...
    />
  </div>

  <span id="description" part="description">
    {this.invalid && this.invalidText && <ds-icon name="alert" />}
    <slot name="description">
      {this.invalid && this.invalidText ? this.invalidText : this.description}
    </slot>
  </span>
</Host>
```

## 3. SCSS Architecture

### `textarea.mixin.scss`

Owns only the container (which differs from input). Delegates everything else to `input.mixin.scss` via wrapper mixins so callers see a uniform interface:

```scss
@use '../input/input.mixin' as input;

// Delegated — identical to input mixins
@mixin vars($prefix)              { @include input.vars($prefix); }
@mixin style($prefix, $sel)       { @include input.style($prefix, $sel); }
@mixin vars-label($prefix)        { @include input.vars-label($prefix); }
@mixin label($prefix, $sel)       { @include input.label($prefix, $sel); }
@mixin vars-description($prefix)  { @include input.vars-description($prefix); }
@mixin description($prefix, $sel) { @include input.description($prefix, $sel); }

// Textarea-specific container — same tokens, different layout
@mixin vars-container($prefix) { @include input.vars-container($prefix); }

@mixin container($prefix, $selector: '#container') {
  #{$selector} {
    // Same border/background/radius/hover/focus as input container
    // Key differences from input:
    align-items: flex-start;   // text anchored to top, not vertically centred
    padding-block: 0.5rem;     // vertical breathing room
    // no min-height — rows prop controls height via native textarea
    textarea { resize: vertical; }
  }
}
```

**CSS variable prefix:** stays `input` throughout. The design tokens are `--ds-input-*` and the textarea shares the same visual token space. No new tokens are needed.

### `textarea.host.scss` (shadow DOM)

```scss
:host {
  @include textarea.vars('input');
  @include textarea.vars-container('input');
  @include textarea.vars-label('input');
  @include textarea.vars-description('input');

  display: grid;
  gap: 0.25rem;
  position: relative;

  @include textarea.style('input', '#textarea');
  @include textarea.container('input');
  @include textarea.label('input');
  @include textarea.description('input');
}

:host(.is-danger)   { /* --mod-input-container-border-color, description-color, hover/active */ }
:host(.is-success)  { /* same pattern */ }
:host(.is-warning)  { /* same pattern */ }
:host(.is-disabled) {
  &, #label, #description, #textarea { pointer-events: none; cursor: default; }
  /* --mod-input-container-background, border, label-color, description-color, color, placeholder */
}
```

Modifier variables follow the same four-layer CSS custom property system used by `ds-input`:
`--_input-*` (private) → `--input-*` (public) → `--mod-input-*` (modifier) → `--ds-input-*` (token).

### `textarea.style.scss` (CSS-only / hybrid mode)

Same structure as `input.style.scss`. Targets `.field`, `.textarea`, `.label`, `.help` class selectors. State variants use `.field.is-danger`, `.field.is-disabled`, etc.

## 4. Tests & Page Object

### `textarea.po.ts`

```ts
export class DsTextarea extends PageObject {
  readonly nativeTextarea: Locator
  constructor(el: E2ELocator) {
    super(el)
    this.nativeTextarea = el.locator('[part="textarea"]')
  }
  async fill(value: string) { await this.nativeTextarea.fill(value) }
  async blur() { await this.nativeTextarea.blur() }
  async assertValue(value: string) { await expect(this.nativeTextarea).toHaveValue(value) }
  async assertToBeDisabled() { await expect(this.nativeTextarea).toBeDisabled() }
}
```

### `textarea.component.play.ts`

Covers:
- `dsInput` fires on fill
- `dsChange` fires with value on blur
- `dsChange` does NOT fire when value unchanged on blur
- disabled native textarea is disabled
- form reset restores initial value

### `textarea.a11y.play.ts`

Mounts each state and calls `a11y('ds-textarea')`:
`default`, `with placeholder`, `with value`, `disabled`, `invalid`, `success`, `warning`

### `textarea.visual.html` sections

| `data-testid` | What it shows |
|---|---|
| `basic` | label + description + placeholder |
| `disabled` | disabled state with value |
| `invalid` | invalid + invalidText |
| `valid` | `color="success"` |
| `warning` | `color="warning"` |
| `rows` | `rows="6"` to verify height control |
| `long-content` | long label + description + overflowing value |

### `textarea.visual.play.ts`

Two describe blocks (`style` and `host`), each loading the respective HTML fixture, one test per section.

### `textarea.style.html`

CSS-only fixture with `.field` / `.textarea` / `.label` / `.help` class selectors, covering `basic`, `disabled`, `invalid`, `valid`, `warning` sections.

## 6. Migration Skill

The Claude skill (`skills/migrate-component.md`) documents this exact process as a reusable checklist, using `bal-textarea` → `ds-textarea` as the canonical reference. It covers:

1. Rename the component directory and all files (drop `bal-` prefix)
2. Update component decorator: add `shadow: true`, `formAssociated: true`, rename tag to `ds-`
3. Replace `FormInput` + old utils with `FormControl` class
4. Remove `BEM` — replace with `id` attributes on elements
5. Rename interfaces namespace (`BalProps`/`BalEvents` → `DS`)
6. Add `label`, `description`, `color`, `invalidText` props
7. Rename events to `ds` prefix
8. Create `component.mixin.scss` (delegate to `input.mixin`, own container)
9. Create `component.host.scss` and `component.style.scss`
10. Rewrite `visual.html` (new sections, `ds-` tags)
11. Create `style.html` (CSS-only fixture)
12. Rewrite `visual.play.ts` (two describe blocks)
13. Create `a11y.play.ts`
14. Create `component.play.ts`
15. Create `component.po.ts` in `packages/playwright`
16. Create Storybook `stories.ts` and `.mdx`
17. Delete old `bal-*` directory
